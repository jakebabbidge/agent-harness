  <name>Task 1: Create Dockerfile and init-firewall.sh</name>
  <files>docker/Dockerfile, docker/init-firewall.sh</files>
  <action>
Create `docker/` directory with two files:

**docker/Dockerfile** -- Based on Claude Code devcontainer reference:
- `FROM node:20` (not alpine -- Claude Code needs full Debian for iptables, ipset, etc.)
- Install system deps: `git, iptables, ipset, iproute2, dnsutils, sudo, curl` via apt-get with `--no-install-recommends` and cleanup
- Install Claude Code CLI globally: `RUN npm install -g @anthropic-ai/claude-code`
- Grant node user passwordless sudo for firewall commands: write to `/etc/sudoers.d/node-firewall` allowing `/usr/sbin/iptables, /usr/sbin/ip6tables, /usr/sbin/ipset, /usr/local/bin/init-firewall.sh`
- COPY init-firewall.sh to `/usr/local/bin/init-firewall.sh` and chmod +x
- WORKDIR /workspace
- USER node

**docker/init-firewall.sh** -- iptables firewall with default-deny and whitelist:
- `#!/usr/bin/env bash` with `set -euo pipefail`
- Flush existing rules: `iptables -F && iptables -X`
- Allow loopback (INPUT and OUTPUT on lo)
- Allow established/related connections (INPUT and OUTPUT)
- Allow DNS (udp+tcp port 53 OUTPUT)
- Create ipset `allowed-domains` as `hash:net` with `-exist`
- Resolve and add IPs for: `api.anthropic.com`, `registry.npmjs.org` using `dig +short`
- Fetch GitHub IP ranges from `https://api.github.com/meta`, parse CIDRs, add to ipset
- Allow OUTPUT to ipset `allowed-domains` via `-m set --match-set`
- Set default deny: `iptables -A OUTPUT -j DROP` and `iptables -P OUTPUT DROP`
- Echo success message at end

Note: The script must be run with `sudo` since the container runs as `node` user but iptables requires root. The Dockerfile grants passwordless sudo for this script.

**`ensureImage(docker: Dockerode): Promise<void>`** -- exported:
1. Check if image exists: `docker.getImage(IMAGE_NAME).inspect()` in try/catch
2. If image exists (no error), return immediately
3. If image doesn't exist (404), build it:
   - Use `child_process.execFileSync('docker', ['build', '-t', IMAGE_NAME, DOCKERFILE_DIR])` for simplicity (synchronous is fine for a one-time build)
   - Actually, prefer `child_process.execFile` (async) wrapped in a Promise for non-blocking behavior
   - Log progress: `console.log('[agent-harness] Building container image agent-harness:latest...')`
   - On build success: `console.log('[agent-harness] Image built successfully.')`
   - On build failure: throw with the stderr output

Why `docker build` CLI instead of `dockerode.buildImage()`: The CLI handles build context (COPY) more simply than streaming tar to the API. The research doc confirms this approach.

Export both `IMAGE_NAME` and `ensureImage`.

**Rework `src/container/manager.ts`:**

Key changes from current implementation:
- Import `IMAGE_NAME` from `./image.js` and `os` for homedir
- Replace `DEFAULT_IMAGE = 'node:20-alpine'` with imported `IMAGE_NAME`
- Remove: `NetworkMode: 'none'`, `ReadonlyRootfs: true`, `Tmpfs` mounts
- Add: `CapAdd: ['NET_ADMIN', 'NET_RAW']`
- Add bind mount: `${os.homedir()}/.claude:/home/node/.claude:ro`

- Set `User: 'node'`, `WorkingDir: '/workspace'`
- Change Cmd from `['sh', '-c', 'sleep infinity']` to:
  `['bash', '-c', 'sudo /usr/local/bin/init-firewall.sh && claude --dangerously-skip-permissions -p "$(cat /workspace/${promptFilePath})"']`
- Keep `AutoRemove: true` (per user decision -- handles normal exit cleanup)
- Keep labels: `'agent-harness': 'true'`, `'agent-harness.task-id': taskId`

Add new method **`waitForExit(taskId: TaskId): Promise<{ StatusCode: number }>`**:
- Get container from registry
- Call `container.wait()` which returns `{ StatusCode }` when the process exits
- Unregister from registry after wait completes (container auto-removes)
- Return the status code

Keep `stopContainer` and `reclaimOrphans` unchanged (they still work with the new model).
