Modify `src/cli/run.ts` to create a BranchTracker and wire worktree lifecycle:

1. **Add imports:**
   ```typescript
   import { BranchTracker } from '../git/tracker.js';
   import { createWorktree, removeWorktree } from '../git/worktree.js';
   import * as os from 'os';
   import * as path from 'path';
   import * as fs from 'fs/promises';
   ```

2. **Create BranchTracker** after generating runId (both workflow and template mode share it):
   ```typescript
   const runId = uuidv4();
   const trackerDir = path.join(os.tmpdir(), 'agent-harness', 'branches');
   await fs.mkdir(trackerDir, { recursive: true });
   const tracker = new BranchTracker(path.join(trackerDir, `${runId}.json`));
   ```

3. **Workflow mode**: Pass tracker and baseBranch to runWorkflow options:
   ```typescript
   const result = await runWorkflow(workflow, executor, {
     runId,
     workflowPath: target,
     tracker,
     baseBranch: 'main',
   });
   ```
   The runner (from Plan 01) handles per-node worktree create/cleanup internally.

4. **Template mode**: Wrap the executor.executeTask call with worktree lifecycle:
   ```typescript
   const taskId = runId.slice(0, 8);
   const worktreeInfo = await createWorktree(options.repo!, taskId, 'main', tracker);
   try {
     const result = await executor.executeTask(
       rendered.rendered,
       worktreeInfo.worktreePath,
       runId,
     );
     // ... existing result handling
   } finally {
     try {
       await removeWorktree(options.repo!, taskId, tracker);
     } catch (cleanupErr) {
       console.warn(`[agent-harness] Failed to clean up worktree: ${cleanupErr}`);
     }
   }
   ```

5. **Move runId creation** to before the if/else branch so both paths share the same runId and tracker. Currently workflow mode creates runId inside the if block -- move it up.

6. **Log the tracker state path** for debugging: `console.log(\`[agent-harness] Branch tracker: ${trackerDir}/${runId}.json\`);`

Important: Keep all existing error handling (API key check, variables parsing, exit codes). The worktree/tracker additions wrap the existing logic, they do not replace it.

Modify `src/cli/resume.ts` to create and load a BranchTracker for resumed workflows:

1. **Add imports:**
   ```typescript
   import { BranchTracker } from '../git/tracker.js';
   import * as os from 'os';
   import * as path from 'path';
   ```

2. **Create and load BranchTracker** after loading run state, using the same runId-based path convention as run.ts:
   ```typescript
   const trackerStatePath = path.join(
     os.tmpdir(), 'agent-harness', 'branches', `${runId}.json`
   );
   const tracker = new BranchTracker(trackerStatePath);
   await tracker.load(); // Restore branch state from crashed run
   ```

3. **Pass tracker to runWorkflow:**
   ```typescript
   const result = await runWorkflow(state.workflowDef, executor, {
     runId,
     state,
     workflowPath: state.workflowPath,
     tracker,
     baseBranch: 'main',
   });
   ```

4. **Handle stale worktrees on resume**: Before calling runWorkflow, iterate over non-completed nodes and attempt cleanup of stale worktrees from the crashed run. This prevents the "branch already exists" error on resume:
   ```typescript
   // Clean up stale worktrees from crashed run before resuming
   const { removeWorktree } = await import('../git/worktree.js');
   for (const node of state.workflowDef.nodes) {
     const ns = state.nodeStates[node.id];
     if (ns && ns.status === 'running') {
       const staleTaskId = `${runId.slice(0, 8)}-${node.id}`;
       try {
         await removeWorktree(node.repo, staleTaskId, tracker);
         console.log(`[agent-harness] Cleaned up stale worktree for node '${node.id}'`);
       } catch {
         // Worktree may not exist if crash happened before creation -- that's fine
       }
     }
   }
   ```

Important: The stale worktree cleanup must happen BEFORE runWorkflow is called. The taskId pattern (`${runId.slice(0, 8)}-${nodeId}`) must match what runner.ts uses (from Plan 01).
