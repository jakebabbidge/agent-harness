Modify `src/workflow/runner.ts` to integrate worktree creation/cleanup around per-node execution:

1. **Add imports** at top of file:
   ```typescript
   import { createWorktree, removeWorktree } from '../git/worktree.js';
   import type { BranchTracker } from '../git/tracker.js';
   ```

2. **Extend RunWorkflowOptions** interface with new optional fields:
   ```typescript
   export interface RunWorkflowOptions {
     runId?: string;
     state?: WorkflowRunState;
     workflowPath?: string;
     stateManager?: ReturnType<typeof createStateManager>;
     tracker?: BranchTracker;    // branch tracking across nodes
     baseBranch?: string;        // branch to base worktrees on (default: "main")
   }
   ```

3. **Extract tracker and baseBranch** at the start of `runWorkflow`:
   ```typescript
   const tracker = options?.tracker;
   const baseBranch = options?.baseBranch ?? 'main';
   ```

4. **Wrap per-node execution** (the async function inside `toExecute.map(...)`, lines 206-249) with worktree lifecycle. The key change is inside the `promises` map callback:

   - Before the try block, if `node.repo` is defined, create a worktree:
     ```typescript
     const worktreeTaskId = `${runId.slice(0, 8)}-${nodeId}`;
     let executionRepo = node.repo;
     let worktreeCreated = false;

     if (node.repo) {
       const worktreeInfo = await createWorktree(node.repo, worktreeTaskId, baseBranch, tracker);
       executionRepo = worktreeInfo.worktreePath;
       worktreeCreated = true;
     }
     ```

   - Change `executor.executeTask(rendered.rendered, node.repo, uuidv4())` to use `executionRepo` instead of `node.repo`.

   - Add a finally block that cleans up the worktree:
     ```typescript
     } finally {
       if (worktreeCreated) {
         try {
           await removeWorktree(node.repo, worktreeTaskId, tracker);
         } catch (cleanupErr) {
           console.warn(`[workflow] Failed to clean up worktree for node '${nodeId}': ${cleanupErr}`);
         }
       }
     }
     ```

   - The existing try/catch for the executor call should be nested inside this outer try/finally. Restructure so the outer try/finally handles worktree cleanup and the inner try/catch handles execution errors + state updates.

5. **Preserve all existing behavior**: When `node.repo` is undefined or empty, skip worktree creation and pass `node.repo` directly (backward compatible). All existing tests must still pass.

Important: The worktree taskId must incorporate both runId (for cross-run uniqueness) and nodeId (for cross-node uniqueness within a run). Use `${runId.slice(0, 8)}-${nodeId}` per the research recommendation.
