<feature>
  <name>Prompt Template Engine</name>
  <files>src/template/renderer.ts, src/template/loader.ts, src/template/renderer.test.ts</files>
  <behavior>
    - TMPL-01: renderTemplate('path/to/file.hbs', { name: 'world' }, []) → rendered string with {{name}} replaced by "world"
    - TMPL-01: renderTemplate with unknown variable → renders empty string for that slot (Handlebars default)
    - TMPL-02: renderTemplate with partialPaths → partial content is inlined at {{> partialName}} locations
    - TMPL-02: two partials with different names both render correctly in one call
    - TMPL-03: dryRunRender(templatePath, variables, partialPaths) → returns same RenderResult as renderTemplate with no observable side effects (no files written, no processes spawned)
    - Concurrency safety: two concurrent renderTemplate calls with different partials do not cross-contaminate (use Handlebars.create() per render call)
    - Error: template file not found → throws Error with descriptive message including the path
    - Error: partial file not found → throws Error with descriptive message
  </behavior>
  <implementation>
    Use `handlebars` package (already in package.json from Plan 01). Key rules:
    - ALWAYS use `Handlebars.create()` for each render call — never the global Handlebars instance — to prevent partial bleed in concurrent use (Pitfall 4 from research).
    - Partial name is derived from filename without extension: `system-prompt.hbs` → partial name `system-prompt`.
    - `dryRunRender` is identical to `renderTemplate` in logic — it IS the dry-run. There are no side effects to suppress; dry-run is just render-without-execute.
    - All file I/O uses `fs/promises` (async). No sync file reads.
    - All relative imports use `.js` extension (NodeNext requirement).
    - Export both `renderTemplate` and `dryRunRender` from renderer.ts. `dryRunRender` can be a re-export or thin wrapper that prints to stdout in addition to returning.

    File structure:
    - `src/template/loader.ts`: `loadTemplate(templatePath): Promise<string>` reads a file, `loadPartials(partialPaths): Promise<Map<string, string>>` reads all partial files returning name→content map.
    - `src/template/renderer.ts`: `renderTemplate(templatePath, variables, partialPaths): Promise<RenderResult>` uses loader + Handlebars.create(). `dryRunRender` is the same but also writes rendered output to stdout.

    Test file `src/template/renderer.test.ts` uses vitest. Use `tmp` directory fixtures or in-memory string writes via `fs/promises.writeFile` to a temp path created with `os.tmpdir()`. Clean up temp files in `afterEach`.
  </implementation>
</feature>
