<feature>
  <name>Git Worktree Isolation + Branch Tracker</name>
  <files>src/git/worktree.ts, src/git/tracker.ts, src/git/worktree.test.ts, src/git/tracker.test.ts</files>
  <behavior>
    GIT-01 (worktree isolation):
    - createWorktree(repoPath, taskId, baseBranch) → creates a new worktree at `<repoPath>/.worktrees/<taskId>/` on branch `agent-harness/task-<taskId>`, returns WorktreeInfo
    - removeWorktree(repoPath, taskId) → removes the worktree dir and deletes the branch; `git worktree list` no longer shows it
    - Two calls to createWorktree with different taskIds produce different worktreePaths
    - Writing a file at `worktreeA/testfile.txt` does NOT cause it to appear at `worktreeB/testfile.txt`
    - If branchName already exists (from a previous run), createWorktree should either fail with a clear error OR use `--force` on the branch; document the chosen behavior

    GIT-02 (branch tracking):
    - BranchTracker.register(info: WorktreeInfo) → stores the mapping in memory
    - BranchTracker.getBranch(taskId) → returns the branchName, or undefined if not registered
    - BranchTracker.getAll() → returns all registered WorktreeInfo entries
    - BranchTracker.unregister(taskId) → removes the entry
    - After process restart (simulated by constructing a new BranchTracker from persisted state file), previously registered tasks are recoverable
  </behavior>
  <implementation>
    `src/git/worktree.ts`:
    - Use `child_process.execFile` with `promisify` — NOT `exec` (execFile is safer; no shell injection). Import as `import { execFile } from 'child_process'`.
    - All relative imports use `.js` extension.
    - worktreePath: `path.join(repoPath, '.worktrees', taskId)` — keep worktrees inside repo root.
    - Branch name: `agent-harness/task-${taskId}` — consistent convention.
    - createWorktree command: `git worktree add -b <branchName> <worktreePath> <baseBranch>` (from cwd=repoPath).
    - removeWorktree: `git worktree remove --force <worktreePath>` then `git worktree prune` then `git branch -d <branchName>`.
    - Wrap execFileAsync errors to include the git stderr output in the thrown Error message.

    `src/git/tracker.ts`:
    - `BranchTracker` class with an in-memory `Map<TaskId, WorktreeInfo>`.
    - Constructor accepts an optional `statePath: string` (path to a JSON state file).
    - `async load()` — loads state from disk if statePath provided and file exists.
    - `async save()` — writes state to disk as JSON if statePath provided.
    - `register`, `getBranch`, `getAll`, `unregister` methods.
    - Persist only after register/unregister calls (call `save()` internally on mutation if statePath set).

    Tests (`worktree.test.ts`):
    - Use `os.tmpdir()` + `crypto.randomUUID()` to create a temp directory per test.
    - In `beforeEach`: `git init` the temp dir to create a real git repo. Add a first commit (git requires at least one commit for branches to work): create a dummy file, `git add .`, `git commit -m "init"`.
    - Test filesystem isolation: create two worktrees, write a file in one, assert it is absent in the other.
    - In `afterEach`: clean up temp directories with `fs.rm(tmpDir, {recursive: true, force: true})`.

    Tests (`tracker.test.ts`):
    - Pure unit tests — no filesystem. Test in-memory register/get/unregister.
    - Test persistence: write to a temp state file, construct a new BranchTracker, call load(), verify data is recovered.
  </implementation>
</feature>
