  <files>package.json, tsconfig.json</files>
  <action>
    Create `package.json` with the following exact shape:

    ```json
    {
      "name": "agent-harness",
      "version": "0.1.0",
      "description": "CLI tool wrapping coding agents with standardised execution workflows",
      "type": "module",
      "bin": {
        "agent-harness": "./dist/bin/agent-harness.js"
      },
      "scripts": {
        "build": "tsc",
        "dev": "tsx watch src/cli/index.ts",
        "test": "vitest run",
        "test:watch": "vitest",
        "typecheck": "tsc --noEmit"
      },
      "dependencies": {
        "commander": "^12.0.0",
        "dockerode": "^4.0.0",
        "handlebars": "^4.7.8",
        "zod": "^3.0.0"
      },
      "devDependencies": {
        "@types/dockerode": "^3.3.0",
        "@types/node": "^20.0.0",
        "tsx": "^4.0.0",
        "typescript": "^5.0.0",
        "vitest": "^2.0.0"
      }
    }
    ```

    Create `tsconfig.json` with the following shape:

    ```json
    {
      "compilerOptions": {
        "module": "NodeNext",
        "moduleResolution": "NodeNext",
        "target": "ES2022",
        "lib": ["ES2022"],
        "outDir": "./dist",
        "rootDir": "./src",
        "strict": true,
        "declaration": true,
        "declarationMap": true,
        "sourceMap": true,
        "esModuleInterop": false,
        "skipLibCheck": true,
        "forceConsistentCasingInFileNames": true
      },
      "include": ["src/**/*"],
      "exclude": ["node_modules", "dist", "**/*.test.ts"]
    }
    ```

    Run `npm install` to install all dependencies.

    **Critical:** `"type": "module"` is mandatory. All relative imports in source files MUST use `.js` extensions (e.g., `import { foo } from './foo.js'`). TypeScript with NodeNext resolves `.ts` source files when `.js` is referenced — this is the correct pattern, not a bug.

    **DO NOT** use `"moduleResolution": "bundler"` or `"module": "ESNext"` — these break Node.js native ESM runtime.
  </action>

    <automated>cd /Users/jake/dev/agent-harness && node -e "import('./package.json', {assert:{type:'json'}}).then(m => { if (m.default.type !== 'module') throw new Error('type must be module'); console.log('ok'); })" 2>/dev/null || node -e "const p = JSON.parse(require('fs').readFileSync('package.json','utf8')); if(p.type !== 'module') throw new Error('not esm'); console.log('ok');"</automated>
  </verify>
  <done>package.json has "type": "module", all deps listed, scripts present. tsconfig.json has module/moduleResolution: NodeNext. node_modules/ populated after npm install.</done>

    Create `src/types/index.ts` with shared types used by all subsystems:

    ```typescript
    /**
     * Shared TypeScript interfaces for the agent-harness.
     * All subsystems (template, git, container) import from here.
     */

    /** Unique identifier for a task run. UUID recommended. */
    export type TaskId = string;

    /** Information about a git worktree created for a task. */
    export interface WorktreeInfo {
      taskId: TaskId;
      worktreePath: string;
      branchName: string;
      baseBranch: string;
      createdAt: Date;
    }

    /** Information about a running Docker container for a task. */
    export interface ContainerInfo {
      taskId: TaskId;
      containerId: string;
      containerName: string;
      repoPath: string;
      startedAt: Date;
    }

    /** Overall state of a task tracked by the harness. */
    export interface TaskState {
      taskId: TaskId;
      status: 'pending' | 'running' | 'completed' | 'failed';
      worktree?: WorktreeInfo;
      container?: ContainerInfo;
      createdAt: Date;
      updatedAt: Date;
    }

    /** Variables passed to template rendering. */
    export type TemplateVariables = Record<string, unknown>;

    /** Result of rendering a template. */
    export interface RenderResult {
      rendered: string;
      templatePath: string;
      partialPaths: string[];
      variables: TemplateVariables;
    }
    ```

    The `src/types/index.ts` file must NOT import from any other source files — it is the leaf node in the dependency tree. All other subsystem files import FROM this file.
  </action>
