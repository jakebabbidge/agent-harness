  <name>Task 1: Build TaskExecutor wrapping Claude Agent SDK with HITL callback</name>
  <files>src/executor/executor.ts, src/executor/executor.test.ts</files>
  <behavior>
    - executeTask(prompt, worktreePath, runId) calls SDK query() and returns TaskResult
    - canUseTool callback fires QuestionStore.askAndWait when toolName is "AskUserQuestion"
    - canUseTool returns { behavior: "allow", updatedInput: input } for all non-AskUserQuestion tools
    - After query() completes, reads RESULT.md from worktreePath; returns empty string if file missing
    - exitCode is 0 when SDK result subtype is "success", 1 otherwise
    - purgeRunDir is called at start of executeTask to prevent stale answer pickup
  </behavior>
  <action>
    1. Install the Claude Agent SDK: `npm install @anthropic-ai/claude-agent-sdk`

    2. Create src/executor/executor.ts:
       - Import `query` from "@anthropic-ai/claude-agent-sdk"
       - Import `QuestionStore` from "../hitl/question-store.js"
       - Import `TaskResult` from "../types/index.js"
       - Export class `TaskExecutor`:
         - Constructor takes `questionStore: QuestionStore`
         - `async executeTask(prompt: string, worktreePath: string, runId: string): Promise<TaskResult>`:
           - Purge run directory first: `this.questionStore.purgeRunDir(runId)`
           - Call `query()` with:
             - prompt
             - options.cwd = worktreePath
             - options.allowedTools = ["Read", "Write", "Edit", "Bash", "Glob", "Grep", "AskUserQuestion"]
             - options.permissionMode = "bypassPermissions"
             - options.allowDangerouslySkipPermissions = true
             - options.canUseTool = async (toolName, input) => { ... }
           - In canUseTool: if toolName === "AskUserQuestion", call `this.questionStore.askAndWait(runId, input as { questions: unknown[] })`, then return `{ behavior: "allow" as const, updatedInput: { ...input, answers } }`
           - For all other tools: return `{ behavior: "allow" as const, updatedInput: input }`
           - Iterate async generator: track last message with "result" — check `message.subtype === "success"` for exitCode 0
           - After loop: read RESULT.md from worktreePath with fs.readFile, catch ENOENT and default to ""
           - Return `{ exitCode, resultText }`

    3. Create src/executor/executor.test.ts:
       - IMPORTANT: Mock the SDK — do NOT make real API calls. Use vitest `vi.mock("@anthropic-ai/claude-agent-sdk")` to mock the `query` export.
       - Create a mock async generator factory that yields configurable messages and captures the options passed to query().
       - Test: executeTask calls query with correct prompt and cwd
       - Test: successful result (subtype "success") returns exitCode 0
       - Test: failed result (subtype "error_during_execution") returns exitCode 1
       - Test: canUseTool callback delegates AskUserQuestion to QuestionStore.askAndWait
       - Test: canUseTool callback returns allow for non-AskUserQuestion tools
       - Test: reads RESULT.md content as resultText after execution
       - Test: returns empty resultText when RESULT.md does not exist
       - Test: purgeRunDir called before query starts

    Use mkdtemp for worktree path fixtures. Write a RESULT.md file in the temp dir for the read test.
    Mock QuestionStore using `vi.fn()` for askAndWait and purgeRunDir.

<task type="auto" tdd="true">
  <name>Task 2: Build sequential WorkflowRunner</name>
  <files>src/workflow/runner.ts, src/workflow/runner.test.ts</files>
  <behavior>
    - runWorkflow(workflow, executor, templateRenderer) executes nodes in array order
    - Each node: renders template with variables, then calls executor.executeTask
    - Stops on first node failure (exitCode !== 0) and returns failure result
    - Returns success result with all node results when all nodes complete
    - Single-node workflow works (degenerate case of sequential)
    - Multi-node workflow executes in order — second node starts only after first completes
  </behavior>
  <action>
    1. Create src/workflow/runner.ts:
       - Import types: WorkflowDef, NodeDef, TaskResult from "../types/index.js"
       - Import renderTemplate from "../template/renderer.js"
       - Import { TaskExecutor } from "../executor/executor.js"
       - Import { v4 as uuidv4 } from "uuid"

       - Export interface `WorkflowResult`:
         - status: "success" | "failed"
         - nodeResults: Array<{ nodeId: string; result: TaskResult }>
         - failedNodeId?: string

       - Export `async function runWorkflow(workflow: WorkflowDef, executor: TaskExecutor): Promise<WorkflowResult>`:
         - Initialize nodeResults array
         - For each node in workflow.nodes (sequential loop):
           - Generate runId = uuidv4()
           - Render template: `const rendered = await renderTemplate(node.template, node.variables ?? {})`
           - Execute: `const result = await executor.executeTask(rendered.rendered, node.repo, runId)`
           - Push { nodeId: node.id, result } to nodeResults
           - If result.exitCode !== 0: return { status: "failed", nodeResults, failedNodeId: node.id }
           - Log: `console.log("[workflow] Node '${node.id}' completed successfully")`
         - Return { status: "success", nodeResults }

    2. Create src/workflow/runner.test.ts:
       - Mock TaskExecutor: create object with executeTask as vi.fn()
       - Mock renderTemplate: use vi.mock for "../template/renderer.js"
       - Test: single-node workflow calls executor once and returns success
       - Test: multi-node workflow calls executor for each node in order
       - Test: executor receives rendered template text (not raw template path)
       - Test: first node failure stops execution — second node is NOT called
       - Test: failure result includes correct failedNodeId
       - Test: all node results are collected in nodeResults array
       - Test: each node gets a unique runId (UUID format)

    IMPORTANT: Mock renderTemplate to return a simple RenderResult with `rendered: "rendered-prompt-for-{nodeId}"`. Mock executor.executeTask to return configurable TaskResult.
  </action>
