<task type="auto" tdd="true">
  <name>Task 1: Extend shared types and build QuestionStore with file-based IPC</name>
  <files>src/types/index.ts, src/hitl/question-store.ts, src/hitl/question-store.test.ts</files>
  <behavior>
    - askAndWait(runId, input) writes question.json to /tmp/agent-harness/runs/{runId}/
    - askAndWait resolves with answer contents when answer.json appears in the same directory
    - answer.json is deleted (consumed) after reading to prevent stale answer pickup
    - Run directory is created if it does not exist (recursive mkdir)
    - submitAnswer(runId, answers) writes answer.json that unblocks the polling loop
    - If answer.json exists before askAndWait starts (stale from previous run), it is NOT consumed — the caller (executor) is responsible for purging run dir on startup
  </behavior>
  <action>
    1. Extend src/types/index.ts with Phase 2 types. Add BELOW existing types (do not modify existing):
       - `QuestionRecord`: { runId: string; questions: Array<{ question: string; header?: string; options?: Array<{ label: string; description?: string }>; multiSelect?: boolean }>; timestamp: string }
       - `AnswerRecord`: { runId: string; answers: Record<string, string>; answeredAt: string }
       - `NodeDef`: { id: string; template: string; repo: string; variables?: Record<string, unknown> }
       - `EdgeDef`: { from: string; to: string }
       - `WorkflowDef`: { version: string; nodes: NodeDef[]; edges: EdgeDef[] }
       - `TaskResult`: { exitCode: number; resultText: string }
       - `RunState`: { runId: string; workflowPath?: string; status: 'running' | 'completed' | 'failed' | 'waiting_for_answer'; startedAt: string; completedAt?: string }

    2. Create src/hitl/question-store.ts:
       - Export class `QuestionStore` with constructor taking optional `baseDir` (default: path.join(os.tmpdir(), 'agent-harness', 'runs'))
       - `runDir(runId)`: returns path.join(baseDir, runId)
       - `async askAndWait(runId: string, input: { questions: unknown[] }): Promise<Record<string, string>>`:
         - mkdir recursive for runDir
         - Write question.json with QuestionRecord shape (runId, questions from input, ISO timestamp)
         - Log to console: "[agent-harness] Agent asking question for run {runId}"
         - Poll answer.json every 500ms (use `await new Promise(r => setTimeout(r, 500))`)
         - When answer.json found: parse, delete file (fs.unlink), return answers record
       - `async submitAnswer(runId: string, answers: Record<string, string>): Promise<void>`:
         - Read question.json to verify a question exists; throw if not found
         - Write answer.json with AnswerRecord shape
       - `async getQuestion(runId: string): Promise<QuestionRecord | null>`:
         - Read question.json if exists, return parsed; return null if not found
       - `async purgeRunDir(runId: string): Promise<void>`:
         - rm -rf the runDir (fs.rm recursive + force)

    3. Create src/hitl/question-store.test.ts:
       - Use mkdtemp for isolated baseDir per test
       - Test: askAndWait writes question.json, submitAnswer writes answer.json, askAndWait resolves with answers
       - Test: submitAnswer throws if no question.json exists
       - Test: getQuestion returns null when no question exists
       - Test: getQuestion returns parsed QuestionRecord when question exists
       - Test: purgeRunDir removes the directory
       - Test: answer.json is deleted after askAndWait consumes it

    IMPORTANT: All imports use .js extensions per project ESM convention (e.g., `import { QuestionRecord } from '../types/index.js'`).
    Use `import * as fs from 'fs/promises'` and `import * as path from 'path'` and `import * as os from 'os'`.
  </action>

  <files>src/workflow/parser.ts, src/workflow/parser.test.ts</files>
  <behavior>
    - parseWorkflow(filePath) reads YAML file, validates with zod, returns typed WorkflowDef
    - Invalid YAML syntax throws an error
    - Valid YAML with missing required fields (e.g., no nodes) throws ZodError
    - Empty nodes array (nodes: []) throws ZodError (min 1 required)
    - Valid workflow with edges parses correctly
    - Valid workflow with no edges defaults edges to []
    - Variables field on a node is optional and defaults to {}
  </behavior>
  <action>
    1. Install yaml and uuid packages: `npm install yaml uuid && npm install -D @types/uuid`

    2. Create src/workflow/parser.ts:
       - Import `parse` from "yaml" and `z` from "zod"
       - Define and export `NodeDefSchema = z.object({ id: z.string(), template: z.string(), repo: z.string(), variables: z.record(z.unknown()).optional().default({}) })`
       - Define and export `EdgeDefSchema = z.object({ from: z.string(), to: z.string() })`
       - Define and export `WorkflowDefSchema = z.object({ version: z.string(), nodes: z.array(NodeDefSchema).min(1), edges: z.array(EdgeDefSchema).default([]) })`
       - Export type `WorkflowDef = z.infer<typeof WorkflowDefSchema>` (this is the runtime-derived type; the one in types/index.ts is the shared interface — they should match)
       - Export `async function parseWorkflow(filePath: string): Promise<WorkflowDef>`:
         - Read file with fs.readFile
         - Parse with yaml `parse()`
         - Validate with `WorkflowDefSchema.parse()`
         - Return validated result

    3. Create src/workflow/parser.test.ts:

       - Test: missing nodes field throws ZodError
       - Test: missing version field throws ZodError
       - Test: variables default to {} when omitted
       - Test: invalid YAML syntax throws error
       - Test: non-existent file throws ENOENT

    IMPORTANT: Use .js import extensions. Import `parse` from "yaml" (external package, no .js extension needed).
