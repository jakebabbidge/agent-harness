  <name>Task 2: DAG topological sort with tier grouping and cycle detection</name>
  <files>src/workflow/dag.ts, src/workflow/dag.test.ts</files>
  <behavior>
    - topologicalTiers(nodeIds: string[], edges: {from, to}[]) returns string[][] (tiers)
    - Single node, no edges -> [[nodeId]]
    - Two independent nodes, no edges -> [[A, B]] (same tier)
    - Linear chain A->B->C -> [[A], [B], [C]]
    - Diamond: A->B, A->C, B->D, C->D -> [[A], [B, C], [D]]
    - Cycle A->B->A throws Error with message containing "circular"
    - Self-loop A->A throws Error with "circular"
    - Node referenced in edge but not in nodeIds throws Error
  </behavior>
  <action>
    1. Create src/workflow/dag.ts implementing `topologicalTiers` using Kahn's algorithm (BFS with in-degree tracking) per the research doc pattern:
       - Build adjacency list and in-degree map from edges
       - Initialize queue with nodes having in-degree 0
       - Process queue in waves: each wave = one tier, decrement in-degree of neighbors
       - After processing, if scheduled count !== total nodes, throw Error about circular dependencies listing unscheduled nodes
       - Validate that all edge.from and edge.to reference known nodeIds, throw if not

    2. Create src/workflow/dag.test.ts with tests for all behaviors listed above. Use descriptive test names. Import only topologicalTiers from dag.ts.
  </action>

  </verify>
