## Integration Gaps (Critical)

### 1. Container Isolation Not Connected (CONT-01, CONT-02)

Phase 1 built `ContainerManager` with Docker lifecycle, network isolation (`NetworkMode: none`), and filesystem restriction (`ReadonlyRootfs: true`). However, Phase 2's `TaskExecutor` calls the Claude Agent SDK directly on the host. No container is ever created during task execution.

**Impact:** The core value proposition of "isolated container execution" is not delivered. Tasks run unrestricted on the host.

### 2. Git Worktree Isolation Not Connected (GIT-01, GIT-02)

Phase 1 built `createWorktree`/`removeWorktree` and `BranchTracker`. However, `cli/run.ts` passes `options.repo` directly to the executor. No worktree is created. Tasks operate directly in the user's repository.

**Impact:** Concurrent tasks would share filesystem state, defeating the isolation guarantee.

### 3. State Persistence Not Connected (WKFL-05)

Phase 3 built `createStateManager` with atomic write persistence and `loadRunState` for resume. However, `cli/run.ts` does not pass a `stateManager` to `runWorkflow()`. The `saveState` calls inside the runner are no-ops (guarded by `if (!sm) return`).

**Impact:** `agent-harness resume <run-id>` will always fail with "No workflow state found" because nothing was saved.

### 4. Dry-Run Render Not Exposed (TMPL-03)

`dryRunRender` is exported from `template/renderer.ts` and tested, but no CLI command exposes it. Users have no way to perform a dry-run.
